## 1. 模块

### 什么是模块？

模块（module）就是一个 `.py` 文件，里面包含了函数、类、变量的定义，可以被其他程序导入使用。

**好处**：
- 代码复用：写一次，处处使用
- 避免命名冲突：不同模块里的同名函数互不干扰
- 方便组织和维护大型项目

### 导入模块

```python
# 导入整个模块
import math
print(math.sqrt(16))   # 4.0
print(math.pi)         # 3.141592653589793

# 只导入模块中的某个函数
from math import sqrt, pi
print(sqrt(16))        # 不需要加 math. 前缀
print(pi)

# 给模块起别名（常用于名字很长的模块）
import numpy as np     # 数据科学常用写法
```

### 常用内置模块

```python
import os           # 操作系统接口（文件/目录操作）
import sys          # Python 解释器接口
import math         # 数学函数
import random       # 随机数
import datetime     # 日期时间
import json         # JSON 数据处理
import re           # 正则表达式
import time         # 时间相关
```

**使用示例**：

```python
import random
# 生成 1~100 之间的随机整数
n = random.randint(1, 100)
print(n)

import datetime
today = datetime.date.today()
print(today)  # 2026-04-13

import json
data = {'name': '小明', 'age': 18}
json_str = json.dumps(data)           # Python → JSON 字符串
obj = json.loads('{"name": "Alice"}') # JSON 字符串 → Python
```

```bash
pip install pandas         # 安装数据分析库

```

> **推荐工具**：Anaconda（科学计算）、Poetry（项目依赖管理）

### 自定义模块

创建 `mymodule.py`：

```python
# mymodule.py

def greet(name):
    print(f'Hello, {name}!')

PI = 3.14159

class Calculator:
    def add(self, a, b):
        return a + b
```

在其他文件中使用：

```python
# main.py
from mymodule import greet, PI, Calculator

greet('Alice')     # Hello, Alice!
print(PI)          # 3.14159
calc = Calculator()
print(calc.add(1, 2))  # 3
```

### `if __name__ == '__main__'`

每个 Python 文件都有 `__name__` 变量：
- 直接运行时，`__name__ == '__main__'`
- 被导入时，`__name__` 是模块名

```python
# mymodule.py
def greet(name):
    print(f'Hello, {name}!')

# 只有直接运行此文件时，才执行下面的测试代码
if __name__ == '__main__':
    greet('Test')   # 被导入时不会执行
```

## 2. 面向对象基础

### 核心思想

面向对象把**数据（属性）**和**操作数据的方法（函数）**封装在一起，称为**类（Class）**。

类是模板，**实例（Instance）**是用模板创建的具体对象。

比喻：
- 类 = 汽车设计图纸
- 实例 = 按图纸造出来的具体汽车
- 属性 = 颜色、品牌、速度
- 方法 = 加速、刹车、转弯

### 定义类

```python
class Student(object):  # class 类名(父类):
    
    def __init__(self, name, age):  # 构造方法，创建实例时自动调用
        self.name = name  # 实例属性
        self.age = age
    
    def greet(self):      # 实例方法
        print(f'我叫 {self.name}，今年 {self.age} 岁')
    
    def study(self, subject):
        print(f'{self.name} 正在学习 {subject}')
```

**说明**：
- `class Student(object)` — `object` 是所有类的根类，可以省略但推荐写上
- `__init__` — 构造方法，名字固定（双下划线开头结尾），创建实例时自动执行
- `self` — 代表实例本身，必须作为第一个参数（调用时 Python 自动传入，不需要手动传）
- `self.属性名` — 实例属性，每个实例独立拥有

### 创建实例

```python
# 创建实例：类名(构造方法的参数)
s1 = Student('Alice', 18)
s2 = Student('Bob', 20)

# 调用方法
s1.greet()       # 我叫 Alice，今年 18 岁
s2.greet()       # 我叫 Bob，今年 20 岁
s1.study('数学') # Alice 正在学习 数学

# 访问属性
print(s1.name)   # Alice
print(s2.age)    # 20
```

### 数据封装

好的类设计应该**隐藏内部实现细节**，只暴露必要的接口：

```python
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.__balance = balance  # __ 前缀表示私有属性，外部无法直接访问
    
    def deposit(self, amount):
        if amount <= 0:
            raise ValueError('存款金额必须大于 0')
        self.__balance += amount
        print(f'存入 {amount} 元，余额：{self.__balance}')
    
    def withdraw(self, amount):
        if amount > self.__balance:
            raise ValueError('余额不足')
        self.__balance -= amount
        print(f'取出 {amount} 元，余额：{self.__balance}')
    
    def get_balance(self):
        return self.__balance

account = BankAccount('Alice', 1000)
account.deposit(500)   # 存入 500 元，余额：1500
account.withdraw(200)  # 取出 200 元，余额：1300

# account.__balance  # ❌ 报错，私有属性无法直接访问
```

> **命名约定**：
> - `_name`：约定为内部使用，外部不建议访问（但技术上可以）
> - `__name`：Python 会自动改名（name mangling），防止外部直接访问

## 3. 继承与多态

### 继承

子类继承父类的所有属性和方法，还可以新增或重写：

```python
class Animal(object):
    def __init__(self, name):
        self.name = name
    
    def run(self):
        print(f'{self.name} 在跑...')
    
    def eat(self):
        print(f'{self.name} 在吃东西')

# Dog 继承 Animal
class Dog(Animal):
    def run(self):  # 重写父类方法（多态）
        print(f'{self.name} 飞奔！')
    
    def bark(self):  # 新增方法
        print(f'{self.name}：汪汪汪！')

# Cat 继承 Animal
class Cat(Animal):
    def run(self):
        print(f'{self.name} 优雅地走...')

dog = Dog('旺财')
cat = Cat('咪咪')

dog.run()   # 旺财 飞奔！（调用重写后的方法）
cat.run()   # 咪咪 优雅地走...
dog.eat()   # 旺财 在吃东西（继承自父类）
dog.bark()  # 旺财：汪汪汪！
```

**继承层次**：

```
object
  └── Animal
        ├── Dog
        └── Cat
```

所有类最终都继承自 `object`。

### 多态

多态（Polymorphism）：同一个接口，不同的类有不同的实现。

```python
def run_twice(animal):
    animal.run()  # 不管传入的是哪种动物，都调用 run()
    animal.run()

run_twice(Dog('旺财'))
# 旺财 飞奔！
# 旺财 飞奔！

run_twice(Cat('咪咪'))
# 咪咪 优雅地走...
# 咪咪 优雅地走...
```

多态的好处：`run_twice()` 函数不需要关心传入的是哪个类，只要有 `run()` 方法就行。

**"开闭原则"**：
- 对扩展开放：可以随时新增 `Animal` 的子类（如 `Horse`、`Fish`）
- 对修改关闭：不需要修改 `run_twice()` 函数

### 鸭子类型（Duck Typing）

Python 是动态语言，不要求严格的继承体系。只要一个对象有 `run()` 方法，就可以传给 `run_twice()`：

```python
class Timer(object):
    def run(self):
        print('计时器启动...')

run_twice(Timer())  # 虽然 Timer 不继承 Animal，也能用！
```

> 鸭子类型：**"如果它走起路来像鸭子，叫起来像鸭子，那它就是鸭子"**

### 多重继承

Python 支持一个类继承多个父类：

```python
class Runnable:
    def run(self):
        print('能跑')

class Flyable:
    def fly(self):
        print('能飞')

class Bat(Runnable, Flyable):  # 蝙蝠既能跑又能飞
    pass

bat = Bat()
bat.run()  # 能跑
bat.fly()  # 能飞
```

> **Mixin 模式**：把额外功能设计成 `Mixin` 类（如 `LogMixin`、`JSONMixin`），通过多重继承组合到需要的类中，是 Python 中常见的设计模式。

### `super()` — 调用父类方法

```python
class Animal:
    def __init__(self, name):
        self.name = name

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)  # 调用父类的 __init__
        self.breed = breed      # 子类新增属性

dog = Dog('旺财', '金毛')
print(dog.name)   # 旺财
print(dog.breed)  # 金毛
```

## 4. 获取对象信息

当不确定对象的类型或有哪些方法时，可以用以下函数查看：

### `type()` — 获取对象类型

```python
>>> type(123)
<class 'int'>
>>> type('hello')
<class 'str'>
>>> type(None)
<class 'NoneType'>

# 判断类型是否相等
>>> type(123) == int
True
>>> type('abc') == str
True
```

### `isinstance()` — 判断是否是某个类型（推荐）

```python
>>> isinstance(123, int)        # True
>>> isinstance('abc', str)      # True
>>> isinstance(dog, Dog)        # True
>>> isinstance(dog, Animal)     # True（dog 是 Dog，Dog 继承自 Animal）

# 可以同时判断多种类型
>>> isinstance(123, (int, float))  # True
>>> isinstance(3.14, (int, float)) # True
```

> `isinstance()` 比 `type()` 更推荐，因为它能处理继承关系。

### `dir()` — 查看对象的所有属性和方法

```python
>>> dir('hello')
['__add__', '__class__', ..., 'capitalize', 'encode', 'find', 'format',
 'join', 'lower', 'replace', 'split', 'strip', 'upper', ...]
```

### `hasattr()`、`getattr()`、`setattr()` — 动态操作属性

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(3, 4)

# 判断是否有某属性
hasattr(p, 'x')       # True
hasattr(p, 'z')       # False

# 获取属性
getattr(p, 'x')       # 3
getattr(p, 'z', 0)    # 0（属性不存在时返回默认值）

# 设置属性
setattr(p, 'z', 5)
print(p.z)            # 5
```

**实际应用**：根据条件动态调用方法

```python
def load_data(source):
    # 判断 source 是否有 read() 方法
    if hasattr(source, 'read'):
        return source.read()
    return None
```

## 5. 实例属性和类属性

### 实例属性

每个实例独立拥有的属性，在 `__init__` 中通过 `self.属性名` 定义：

```python
class Student:
    def __init__(self, name):
        self.name = name  # 实例属性

s1 = Student('Alice')
s2 = Student('Bob')
print(s1.name)  # Alice
print(s2.name)  # Bob（各自独立）
```

### 类属性

类本身的属性，所有实例共享：

```python
class Student:
    school = '北京大学'  # 类属性

    def __init__(self, name):
        self.name = name   # 实例属性

s1 = Student('Alice')
s2 = Student('Bob')
print(Student.school)  # 北京大学
print(s1.school)       # 北京大学（实例可以访问类属性）
print(s2.school)       # 北京大学
```

**类属性应用：统计实例数量**

```python
class Student:
    count = 0  # 类属性，统计学生数量

    def __init__(self, name):
        self.name = name
        Student.count += 1  # 每创建一个实例，count +1

s1 = Student('Alice')
s2 = Student('Bob')
s3 = Student('Carol')
print(Student.count)  # 3
```

> ⚠️ **不要用实例属性遮盖类属性**：`s1.school = 'xxx'` 会创建一个实例属性 `school`，遮盖类属性，但类属性本身不变。

## 6. 面向对象高级特性

### `__slots__` — 限制实例属性

```python
class Student:
    __slots__ = ('name', 'age')  # 只允许绑定 name 和 age

s = Student()
s.name = 'Alice'  # ✅
s.age = 18        # ✅
s.score = 99      # ❌ AttributeError
```

用途：节省内存（特别是大量实例时）、防止意外添加属性。

### `@property` — 让方法像属性一样访问

直接暴露属性有风险（无法验证值），用 getter/setter 方法又太麻烦。`@property` 完美结合两者：

```python
class Student:
    @property
    def score(self):
        return self._score
    
    @score.setter
    def score(self, value):
        if not isinstance(value, int):
            raise ValueError('成绩必须是整数')
        if not 0 <= value <= 100:
            raise ValueError('成绩必须在 0~100 之间')
        self._score = value

s = Student()
s.score = 85       # 像操作属性一样，实际调用 setter
print(s.score)     # 85，实际调用 getter
s.score = 999      # ValueError: 成绩必须在 0~100 之间
```

**只读属性**（只定义 getter，不定义 setter）：

```python
class Circle:
    def __init__(self, radius):
        self.radius = radius
    
    @property
    def area(self):  # 只读，不提供 setter
        return 3.14159 * self.radius ** 2

c = Circle(5)
print(c.area)   # 78.53975
c.area = 100    # ❌ AttributeError: can't set attribute
```

### 定制类（魔法方法）

Python 用特殊方法（`__xxx__`）让类能响应内置操作：

```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __str__(self):          # print() 时调用
        return f'Vector({self.x}, {self.y})'
    
    def __repr__(self):         # 交互式环境直接输出时调用
        return f'Vector({self.x}, {self.y})'
    
    def __add__(self, other):   # + 运算符
        return Vector(self.x + other.x, self.y + other.y)
    
    def __len__(self):          # len() 函数
        return int((self.x**2 + self.y**2) ** 0.5)

v1 = Vector(3, 4)
v2 = Vector(1, 2)
print(v1)          # Vector(3, 4)
print(v1 + v2)     # Vector(4, 6)
print(len(v1))     # 5（根号 9+16）
```

**常用魔法方法**：

| 方法 | 触发时机 |
|------|----------|
| `__init__` | 创建实例 |
| `__str__` | `print(obj)` |
| `__repr__` | 交互式输出 |
| `__len__` | `len(obj)` |
| `__add__` | `obj1 + obj2` |
| `__eq__` | `obj1 == obj2` |
| `__lt__` | `obj1 < obj2` |
| `__iter__` | `for x in obj` |
| `__getitem__` | `obj[key]` |

## 7. 错误处理

### try...except...finally

```python
try:
    # 可能出错的代码
    x = int(input('请输入数字：'))
    result = 100 / x
    print(f'结果：{result}')
except ValueError:
    # 捕获特定异常
    print('输入的不是有效数字')
except ZeroDivisionError:
    print('不能除以 0')
except Exception as e:
    # 捕获所有异常（兜底）
    print(f'发生未知错误：{e}')
else:
    # 没有异常时执行
    print('计算成功！')
finally:
    # 无论是否出错，都会执行
    print('程序执行完毕')
```

### 异常继承体系

```
BaseException
  └── Exception
        ├── ValueError        # 值错误
        ├── TypeError         # 类型错误
        ├── ZeroDivisionError # 除零错误
        ├── IndexError        # 索引越界
        ├── KeyError          # 字典键不存在
        ├── FileNotFoundError # 文件不存在
        ├── AttributeError    # 属性不存在
        └── ...（很多）
```

### 主动抛出异常

```python
def divide(a, b):
    if b == 0:
        raise ValueError('除数不能为 0')  # 主动抛出异常
    return a / b

try:
    result = divide(10, 0)
except ValueError as e:
    print(f'错误：{e}')  # 错误：除数不能为 0
```

### 自定义异常

```python
class AgeError(ValueError):
    """年龄不合法时抛出"""
    pass

def set_age(age):
    if not 0 < age < 150:
        raise AgeError(f'年龄 {age} 不合法')
    return age

try:
    set_age(200)
except AgeError as e:
    print(f'自定义异常：{e}')
```

### 错误处理的最佳实践

```python
# ✅ 只捕获预期的异常，不要一股脑捕获所有异常
try:
    data = json.loads(text)
except json.JSONDecodeError:
    data = {}

# ❌ 不要这样写（会吞掉所有错误，难以调试）
try:
    ...
except:
    pass
```

## 8. 调试与测试

### 使用 `print()` 调试

最简单粗暴的方法：

```python
def calc(x):
    print(f'[DEBUG] 输入 x = {x}')  # 调试输出
    result = x * 2
    print(f'[DEBUG] 结果 = {result}')
    return result
```

### 使用 `assert` 断言

断言是"我保证这里的条件一定为真"，如果不为真则抛出 `AssertionError`：

```python
def calc(x):
    assert x > 0, f'x 必须大于 0，但得到了 {x}'
    return x * 2

calc(-1)  # AssertionError: x 必须大于 0，但得到了 -1
```

### 使用 logging 记录日志

比 print 更专业：

```python
import logging

logging.basicConfig(level=logging.DEBUG,
                    format='%(levelname)s: %(message)s')

def calc(x):
    logging.debug(f'输入 x = {x}')
    result = x * 2
    logging.info(f'计算完成，结果 = {result}')
    return result

calc(5)
# DEBUG: 输入 x = 5
# INFO: 计算完成，结果 = 10
```

日志级别（由低到高）：`DEBUG < INFO < WARNING < ERROR < CRITICAL`

生产环境可以把级别设为 `WARNING`，过滤掉 DEBUG 和 INFO 信息。

### 单元测试

使用内置的 `unittest` 模块：

```python
# test_calc.py
import unittest

def add(a, b):
    return a + b

class TestAdd(unittest.TestCase):
    def test_positive(self):
        self.assertEqual(add(1, 2), 3)
    
    def test_negative(self):
        self.assertEqual(add(-1, -1), -2)
    
    def test_zero(self):
        self.assertEqual(add(0, 5), 5)
    
    def test_type_error(self):
        with self.assertRaises(TypeError):
            add('a', 1)

if __name__ == '__main__':
    unittest.main()
```

运行测试：

```bash
python -m unittest test_calc.py
# ..
# Ran 3 tests in 0.001s
# OK
```
