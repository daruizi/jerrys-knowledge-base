# 第 10 章：开发者指南

## 插件系统

### 概述

Hermes Agent 支持三种插件发现源：

| 来源 | 位置 | 范围 |
|------|------|------|
| 用户插件 | `~/.hermes/plugins/` | 当前用户 |
| 项目插件 | `.hermes/plugins/` | 当前项目 |
| PIP 入口点 | pip 安装的包 | 全局 |

### 插件能力

插件可以注册：
- **新工具**：扩展 Agent 的能力
- **Hook**：事件驱动的回调
- **CLI 命令**：新增子命令

### 专用插件类型

| 类型 | 位置 | 说明 |
|------|------|------|
| Memory Provider | `plugins/memory/` | 自定义记忆后端 |
| Context Engine | `plugins/context_engine/` | 自定义上下文处理 |

### 构建插件

```python
# 最小插件示例
from hermes.plugins import Plugin

class MyPlugin(Plugin):
    name = "my-plugin"
    version = "1.0.0"

    def activate(self, ctx):
        # 注册工具
        ctx.register_tool(MyCustomTool())
        # 注册 Hook
        ctx.register_hook("pre_tool_call", self.my_hook)
```

## 添加工具

### 开发内置工具

在 Hermes 源码中添加新工具：

```python
# tools/my_new_tool.py
from tools.registry import registry
from tools.base import BaseTool


class MyNewTool(BaseTool):
    name = "my_new_tool"
    description = "工具的描述"

    # 参数定义
    parameters = {
        "type": "object",
        "properties": {
            "param1": {
                "type": "string",
                "description": "参数说明",
            }
        },
        "required": ["param1"],
    }

    async def execute(self, params):
        param1 = params.get("param1")
        # 执行逻辑
        result = do_something(param1)
        return result


registry.register(MyNewTool())
```

### 工具注册机制

```
tools/registry.py  (无依赖)
       ↑
tools/*.py         (import 时自动注册)
       ↑
model_tools.py     (导入注册表并触发发现)
       ↑
run_agent.py 等    (使用工具)
```

关键：工具注册发生在 **import 时**，无需手动维护导入列表。

## 添加 Provider

### Provider 注册

```python
# hermes_cli/auth.py
PROVIDER_REGISTRY = {
    "my_provider": {
        "api_mode": "chat_completions",
        "env_vars": ["MY_PROVIDER_API_KEY"],
        "default_model": "my-model",
    }
}
```

### 运行时解析

Provider 运行时解析器将 `(provider, model)` 映射到具体的 API 调用方式：

```python
# hermes_cli/runtime_provider.py
def resolve_runtime_provider(provider, model):
    return api_mode, api_key, base_url
```

## 代码贡献

### 贡献流程

1. Fork 仓库
2. 创建分支 (`git checkout -b feature/my-feature`)
3. 提交修改 (`git commit -m "feat: add my feature"`)
4. 推送到 Fork (`git push origin feature/my-feature`)
5. 创建 Pull Request

### 提交信息规范

```
feat: 新功能
fix: 修复 bug
docs: 文档更新
style: 代码格式
refactor: 重构
test: 测试相关
chore: 构建/工具链
```
