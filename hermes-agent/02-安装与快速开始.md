# 第 02 章：安装与快速开始

## 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Linux / macOS / WSL2 / Android (Termux) |
| Git | 必须安装 |
| Python/Node 等 | 安装器自动检测并安装 |
| Windows | 不原生支持，需通过 WSL2 |

:::warning
**原生 Windows 不受支持**。请先安装 [WSL2](https://learn.microsoft.com/en-us/windows/wsl/install)，然后在 WSL2 终端中运行安装命令。
:::

---

## 一行安装

### Linux / macOS / WSL2 / Termux

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

安装器自动处理：
- Python 3.11（通过 uv，无需 sudo）
- Node.js v22（浏览器自动化和 WhatsApp 桥接）
- ripgrep（快速文件搜索）
- ffmpeg（TTS 音频格式转换）
- 仓库克隆、虚拟环境、全局 `hermes` 命令

### Android / Termux

同一命令，安装器自动检测 Termux 并切换 Android 流程：
- 使用 Termux `pkg` 安装系统依赖
- 自动导出 `ANDROID_API_LEVEL`
- 安装 `.[termux]` 额外依赖

## 安装后目录结构

| 安装方式 | 代码位置 | hermes 命令 | 数据目录 |
|---------|---------|------------|---------|
| 普通用户 | `~/.hermes/hermes-agent/` | `~/.local/bin/hermes` | `~/.hermes/` |
| Root 模式 | `/usr/local/lib/hermes-agent/` | `/usr/local/bin/hermes` | `/root/.hermes/` |

---

## 启动 Hermes

```bash
source ~/.bashrc   # 或 source ~/.zshrc
hermes             # 开始对话！
```

## 选择 Provider（最重要的一步）

```bash
hermes model
```

### 推荐 Provider

| Provider | 说明 | 设置方式 |
|----------|------|---------|
| **Nous Portal** | 订阅制，零配置 | OAuth 登录 |
| **OpenRouter** | 多 Provider 路由 | 输入 API Key |
| **Anthropic** | Claude 模型 | OAuth（Max 计划）或 API Key |
| **OpenAI Codex** | ChatGPT OAuth | 设备码认证 |
| **Z.AI** | GLM/智谱模型 | 设置 `GLM_API_KEY` |
| **Kimi/Moonshot** | 月之暗面模型 | 设置 `KIMI_API_KEY` |
| **Kimi 中国** | 中国区域月之暗面 | 设置 `KIMI_CN_API_KEY` |
| **阿里 DashScope** | Qwen 系列 | 设置 `DASHSCOPE_API_KEY` |
| **Hugging Face** | 20+ 开源模型 | 设置 `HF_TOKEN` |
| **Ollama** | 本地模型 | 设置 base URL |
| **自定义端点** | VLLM/SGLang/OpenAI 兼容 | 设置 URL + Key |

:::caution 最小上下文窗口
Hermes Agent 需要模型至少支持 **64,000 token** 上下文。上下文窗口小于 64K 的模型无法启动多步工具调用工作流。
:::

### 配置保存位置

```
~/.hermes/.env         ← 密钥和 Token
~/.hermes/config.yaml  ← 非密钥配置
```

## 核心命令速查

| 命令 | 功能 |
|------|------|
| `hermes` | 开始对话 |
| `hermes model` | 选择 LLM Provider 和模型 |
| `hermes tools` | 配置各平台启用的工具 |
| `hermes setup` | 完整设置向导 |
| `hermes doctor` | 诊断问题 |
| `hermes update` | 更新到最新版本 |
| `hermes gateway` | 启动消息网关 |
| `hermes gateway setup` | 消息平台交互式配置 |
| `hermes sessions list` | 查看历史会话 |
| `hermes --continue` | 恢复上次会话 |

### 对话中的斜杠命令

| 命令 | 功能 |
|------|------|
| `/help` | 显示所有可用命令 |

| `/personality pirate` | 尝试不同人格 |

## 常见故障排除

| 症状 | 可能原因 | 修复方法 |
|------|---------|---------|
| 空回复/错误回复 | Provider 认证或模型选择有误 | 重新运行 `hermes model` |
| 自定义端点返回乱码 | 端点不兼容 OpenAI 格式 | 先用其他客户端验证端点 |
| 网关启动但无法接收消息 | Bot Token/白名单设置不完整 | 重新运行 `hermes gateway setup` |
| 找不到旧会话 | 切换了 Profile 或会话未保存 | 检查 `hermes sessions list` |
| 模型不可用 | Provider 路由配置过于激进 | 关闭路由，先稳定基础 Provider |

### 恢复工具箱

当感觉哪里不对时，按顺序执行：

```bash
1. hermes doctor        # 诊断
2. hermes model         # 确认模型
3. hermes setup         # 完整设置
4. hermes sessions list # 查看会话
5. hermes --continue    # 恢复会话
6. hermes gateway status # 检查网关
```

## 多行输入

- `Alt+Enter` 或 `Ctrl+J`：换行
- 适合粘贴代码或编写详细 Prompt

## 中断 Agent

- 输入新消息并回车：中断当前任务，切换到新指令
- `Ctrl+C`：直接中断
