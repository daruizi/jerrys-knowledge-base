# 第 06 章：工具系统

## 概述

Hermes Agent 内置 **61 个工具**，按 **52 个工具集**分组。每个工具在 import 时自动注册到中央注册表。

---

## 工具分类

### 核心工具

| 类别 | 工具 | 说明 |
|------|------|------|
| 文件操作 | read_file, write_file, patch_file, search_files | 读写搜索文件 |
| 终端执行 | terminal | 运行 shell 命令 |
| 网络工具 | web_search, web_extract | 搜索和提取网页内容 |
| 浏览器 | browser_navigate, browser_click, browser_fill 等 10 个工具 | 浏览器自动化 |
| 代码执行 | execute_code | Python 沙箱执行 |
| 子 Agent | delegate | 委派任务给子 Agent |

### 高级工具

| 类别 | 说明 |
|------|------|
| MCP 工具 | 连接外部 MCP 服务器 |
| 图像生成 | 根据文字生成图片 |
| 视觉 | 分析和理解图片 |
| TTS | 文字转语音 |
| 语音 | 语音识别和交互 |
| 凭证管理 | 文件式凭据传递 |

---

## 工具集（Toolsets）

工具集是工具的逻辑分组，便于按平台和场景管理：

```yaml
# config.yaml
toolsets:
  cli:          # CLI 平台默认启用
    enabled: [terminal, file, web, browser, code_execution, delegate]
  telegram:     # Telegram 平台
    enabled: [terminal, file, web, image_generation]
  discord:      # Discord 平台
    enabled: [terminal, file, web, voice]
```

## 终端后端（7 种）

Hermes Agent 的工具执行后端高度灵活：

| 后端 | 说明 | 适用场景 |
|------|------|---------|
| **local** | 直接在宿主机执行 | 本地开发 |
| **docker** | Docker 容器隔离 | 安全执行不信任代码 |
| **ssh** | 远程服务器执行 | 运维、远程管理 |
| **daytona** | Daytona Serverless | 云端按需执行 |
| **modal** | Modal Serverless | 云端弹性执行 |
| **singularity** | HPC 环境 | 高性能计算 |
| **vercel-sandbox** | Vercel 云沙箱 | 云端隔离执行 |

### 切换终端后端

```bash
hermes config set terminal.backend docker    # Docker 隔离
hermes config set terminal.backend ssh       # 远程服务器
hermes config set terminal.backend modal     # Serverless
```

## MCP 集成（Model Context Protocol）

MCP 允许连接外部工具服务器：

```yaml
# config.yaml
mcp_servers:
  github:
    command: npx
    args: ["-y", "@modelcontextprotocol/server-github"]
    env:
      GITHUB_PERSONAL_ACCESS_TOKEN: "ghp_xxx"
  filesystem:
    command: npx
    args: ["-y", "@modelcontextprotocol/server-filesystem"]
    config:
      allowed_directories: ["/path/to/project"]
```

### MCP 工具过滤

可以按名称过滤 MCP 暴露的工具：

```yaml
mcp_servers:
  github:
    command: npx
    args: ["-y", "@modelcontextprotocol/server-github"]
    allowed_tools: ["create_issue", "list_pull_requests"]    # 白名单
    denied_tools: ["delete_repository"]                       # 黑名单
```

## 工具开发指南

添加工具的基本步骤：

1. 在 `tools/` 目录创建新文件
2. 继承 `TerminalTool` 基类
3. 实现 `name`、`description`、`execute` 方法
4. 在文件顶部调用 `registry.register()`

```python
from tools.registry import registry

class MyTool(TerminalTool):
    name = "my_tool"
    description = "执行某个特定任务"

    async def execute(self, params):
        # 实现逻辑
        return result

registry.register(MyTool())
```
