# 第 11 章：实用场景与案例

## 社区真实用例

以下内容来源于 X、GitHub、Reddit、Hacker News、YouTube、博客和播客的社区反馈：

---

### 场景 1：始终在线的个人助手

**需求：** 在 $5 VPS 上部署，通过 Telegram 随时对话，有跨会话记忆。

**配置：**
```bash
# 在 VPS 上安装
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

# 配置 Telegram
hermes gateway setup
# 选择 Telegram，粘贴 Bot Token

# 启动
hermes gateway start

# 使用 Serverless 后端节省成本
hermes config set terminal.backend modal
```

**效果：**
- 随时随地通过 Telegram 与 AI 对话
- Agent 记住之前的项目和偏好
- Modal 后端在闲置时几乎零成本

---

### 场景 2：自动化多步工作流

**需求：** 每天早上 8 点自动获取新闻摘要并发送到 Telegram。

**配置：**
```
/cron set "0 8 * * *" "获取今日科技新闻摘要，总结 5 条最重要的"
```

**效果：** 每天早上自动收到个性化新闻摘要，无需手动操作。

---

### 场景 3：GitHub PR 审查 Agent

**需求：** 自动审查 Pull Request，给出代码质量反馈。

**配置：**
1. 配置 GitHub MCP 服务器
2. 安装 PR 审查技能
3. 通过 Webhook 自动触发

```yaml
# Webhook 配置
mcp_servers:
  github:
    command: npx
    args: ["-y", "@modelcontextprotocol/server-github"]

# 技能
skills:
  telegram:
    enabled: [pr-reviewer]
```

**效果：** 每次 PR 提交后自动触发代码审查，结果发送到 Telegram 群组。

---

### 场景 4：团队 Telegram 助手

**需求：** 团队共用一个 AI 助手，白名单控制访问。

**配置：**
```yaml
gateway:
  telegram:
    allowlist: ["user1_id", "user2_id", "user3_id"]
    dm_pairing: true
    admin_ids: ["admin_id"]

toolsets:
  telegram:
    enabled: [terminal, file, web, delegate]
```

**效果：** 团队成员共享一个智能助手，管理员可管理，成员可提问。

---

### 场景 5：本地模型部署

**需求：** 使用本地 Ollama 运行开源模型，保护隐私。

**配置：**
```bash
# 启动 Ollama
ollama run qwen2.5-coder:32b --ctx-size 65536

# 配置 Hermes
hermes model
# 选择 Custom Endpoint
# Base URL: http://localhost:11434/v1
# Model: qwen2.5-coder:32b
```

:::caution
确保本地模型的上下文窗口 ≥ 64K token，否则无法正常工作。
:::

## 最佳实践

### 学习顺序

```
安装 → 基本对话 → 配置 Provider → 验证会话恢复
→ 探索工具 → 设置消息平台 → 配置记忆/技能
→ 定时任务 → 子 Agent → 自定义工具/技能
```

### 推荐模型

| 用途 | 推荐模型 | 原因 |
|------|---------|------|
| 通用对话 | Claude Opus / GPT-4o | 理解能力强 |
| 代码 | Claude Sonnet / GPT-4o | 代码能力优秀 |
| 本地 | Qwen 2.5 Coder 32B | 开源最强编码模型 |
| 性价比 | OpenRouter 路由 | 自动选择最优 Provider |

### 安全建议

1. **使用 Docker 后端**执行不信任代码
2. **配置白名单**限制消息平台访问
3. **禁用危险工具**在生产环境中
4. **保护 API Key**存放在 `.env` 文件中
5. **定期更新**保持最新安全补丁

## 常见问题 FAQ

### Q: Hermes Agent 和 ChatGPT/Claude 有什么区别？

A: ChatGPT/Claude 是对话机器人，每次对话从零开始。Hermes Agent 是一个**自主 Agent**，有记忆、能学习、可自动化执行各种工具。

### Q: 需要什么硬件配置？

A: 本地运行最低 8GB RAM（推荐 16GB）。如果使用云端 Provider（OpenRouter 等），几乎任何设备都可以（包括手机）。

### Q: 支持 Windows 吗？

A: 不支持原生 Windows。需要安装 WSL2，然后在 WSL2 中运行。

### Q: 如何更新到最新版本？

```bash
hermes update
```

### Q: 可以同时运行多个 Profile 吗？

A: 可以。每个 Profile 是独立的实例，可并发运行：

```bash
hermes -p personal start &
hermes -p work start &
```
