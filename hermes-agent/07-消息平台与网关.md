# 第 07 章：消息平台与网关

## 概述

Hermes Agent 的**消息网关**是一个常驻进程，将 AI Agent 连接到各种消息平台。用户从 Telegram/Discord/Slack 等平台发送消息 → 网关路由到 Agent → Agent 处理并返回结果。

## 快速接入 Telegram（推荐）

### 1. 创建 Telegram Bot

1. 在 Telegram 中搜索 `@BotFather`
2. 发送 `/newbot` 命令
3. 按提示设置 Bot 名称
4. 获取 Bot Token（格式：`123456:ABC-DEF...`）

### 2. 配置 Hermes

```bash
hermes gateway setup
# 选择 Telegram
# 粘贴 Bot Token
```

### 3. 启动网关

```bash
hermes gateway start
```

### 4. 验证

在 Telegram 中给你的 Bot 发消息，应该能收到 AI 回复。

---

## Discord 接入

### 1. 创建 Discord Application

1. 访问 [Discord Developer Portal](https://discord.com/developers/applications)
2. 创建新 Application
3. 在 "Bot" 页面创建 Bot
4. 复制 Bot Token

### 2. 配置

```bash
hermes gateway setup
# 选择 Discord
# 粘贴 Bot Token
# 设置允许的频道/服务器
```

### 3. 邀请 Bot 到服务器

使用 OAuth2 URL 邀请 Bot 到你的 Discord 服务器。

## 网关管理

### 启动/停止

```bash
hermes gateway start      # 启动网关
hermes gateway stop       # 停止网关
hermes gateway restart    # 重启网关
hermes gateway status     # 查看状态
```

### 用户授权

网关支持用户白名单和 DM 配对：

```yaml
# config.yaml
gateway:
  telegram:
    allowlist: ["user_id_1", "user_id_2"]   # 允许的用户
    dm_pairing: true                         # 允许私聊
```

## 平台隔离

每个平台独立的会话、工具和技能配置：

```yaml
# 不同平台使用不同的工具集
toolsets:
  cli:
    enabled: [terminal, file, web, browser, code_execution, delegate]
  telegram:
    enabled: [terminal, file, web]           # 消息平台精简版
  discord:
    enabled: [terminal, file, web, voice]    # 带语音
```

---

## Profile 多实例

```bash
hermes -p personal              # 个人 Profile
hermes -p work                  # 工作 Profile
hermes -p dev --profile dev     # 开发 Profile
```

每个 Profile 有独立的：
- `HERMES_HOME` 目录
- 配置文件
- 记忆
- 会话
- 网关 PID

多个 Profile 可以**同时运行**。

---

## 安全最佳实践

| 实践 | 说明 |
|------|------|
| 白名单 | 只允许特定用户/群组使用 |
| DM 配对 | 谨慎开启私聊模式 |
| 工具限制 | 消息平台限制可用的工具 |
| Docker 后端 | 使用容器隔离执行命令 |
| 命令审批 | 高危命令需要确认 |
