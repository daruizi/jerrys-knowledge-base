# 第 08 章：配置与个性化

## 配置文件结构

Hermes Agent 将配置分为两类：

| 文件 | 内容 | 位置 |
|------|------|------|
| `config.yaml` | 普通配置（模型、工具、平台等） | `~/.hermes/config.yaml` |
| `.env` | 密钥和 Token | `~/.hermes/.env` |

---

## 常用配置命令

```bash
hermes config set model anthropic/claude-opus-4.6      # 设置模型
hermes config set terminal.backend docker               # 设置终端后端
hermes config set OPENROUTER_API_KEY sk-or-...          # 设置 API Key
hermes config get model                                 # 查看当前模型
hermes config check                                     # 检查配置
hermes config migrate                                   # 迁移旧配置
```

---

## 模型配置

### 方式一：交互式

```bash
hermes model
# 按提示选择 Provider 和模型
```

### 方式二：直接设置

```bash
hermes config set model anthropic/claude-opus-4.6
hermes config set model openrouter/openai/gpt-4o
hermes config set model openai/gpt-4o
```

### 方式三：环境变量

```bash
export OPENROUTER_API_KEY=sk-or-...
export ANTHROPIC_API_KEY=sk-ant-...
```

## 人格系统（SOUL.md）

Hermes Agent 支持自定义人格，通过 `SOUL.md` 文件定义：

### 设置人格

```bash
# 对话中
/personality pirate          # 内置人格
/personality philosopher     # 哲学家
/personality helpful         # 友好助手
/personality default         # 恢复默认
```

### 自定义人格

创建 `~/.hermes/SOUL.md`：

```markdown
# SOUL.md

你是一个资深技术顾问，风格专业但友好。
- 回答要简洁有力，避免冗长
- 优先给出可执行的建议
- 遇到不确定的问题时坦诚说"不确定"
- 适当使用技术术语，但会解释
```

Agent 启动时自动加载全局 `SOUL.md` 作为默认人格。

## 上下文文件（Context Files）

上下文文件是让 Agent 了解项目/工作环境的文件。

### 项目级上下文

在项目根目录创建 `.hermes.md`：

```markdown
# .hermes.md

## 项目信息
- 名称：my-project
- 框架：FastAPI + React
- Python 版本：3.11

## 代码规范
- 使用 Black 格式化
- 函数不超过 50 行
- 所有 API 端点需要类型注解
```

Agent 进入该项目目录时自动加载此文件。

### 目录级上下文

在任何目录放置 `AGENTS.md`：

```markdown
# AGENTS.md

本目录包含前端组件：
- `components/` — React 组件
- `hooks/` — 自定义 Hooks
- `types/` — TypeScript 类型定义
```

## Profile 配置

Profile 是独立的运行环境，每个 Profile 有独立的配置：

```bash
hermes -p personal start       # 使用 personal profile
hermes -p work start           # 使用 work profile
hermes profiles list           # 列出所有 profile
hermes profiles create dev     # 创建新 profile
hermes profiles delete dev     # 删除 profile
```

## 主题/皮肤

```bash
# 对话中
/skins list                    # 列出可用主题
/skins set dark                # 切换深色主题
/skins set light               # 切换亮色主题
/skins set neon                # 霓虹主题
```

## 配置示例

```yaml
# ~/.hermes/config.yaml 示例

model: anthropic/claude-opus-4.6

terminal:
  backend: docker
  timeout: 300

gateway:
  telegram:
    token: "${TELEGRAM_BOT_TOKEN}"
    allowlist: ["123456789"]

tools:
  enabled: [terminal, file, web, browser]

skills:
  cli:
    enabled: [code-review, git-helper]

mcp_servers:
  github:
    command: npx
    args: ["-y", "@modelcontextprotocol/server-github"]
    env:
      GITHUB_PERSONAL_ACCESS_TOKEN: "${GITHUB_TOKEN}"

memory:
  provider: sqlite
```
