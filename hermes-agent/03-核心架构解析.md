## 核心组件

### AIAgent — 大脑

`run_agent.py` 中的 `AIAgent` 类（约 13,700 行），是整个系统的核心：

- **Provider 选择**：根据配置动态选择 LLM Provider
- **Prompt 构建**：组合人格、记忆、技能、上下文文件等
- **工具调度**：解析模型返回的 tool_calls，分发到对应工具
- **会话管理**：自动保存/恢复对话历史
- **压缩与缓存**：上下文超限时的摘要压缩 + Anthropic prompt caching
- **三种 API 模式**：`chat_completions` / `codex_responses` / `anthropic_messages`

### HermesCLI — 交互界面

`cli.py`（约 11,500 行），提供两种终端界面：

| 界面 | 启动命令 | 特点 |
|------|---------|------|
| 经典 CLI | `hermes` | 基于 prompt_toolkit，简洁 |
| TUI | `hermes --tui` | 模态覆盖、鼠标选择、非阻塞输入 |

### 工具系统

```
tools/registry.py          ← 核心注册表（无依赖）
       ↑
tools/*.py                 ← 每个工具文件在 import 时自动注册
       ↑
model_tools.py             ← 工具发现和调度
```

- **61 个已注册工具**
- **52 个工具集**（按平台预设分组）
- **7 种终端后端**：local, docker, ssh, daytona, modal, singularity, vercel-sandbox

### 消息网关

`gateway/run.py`（约 12,200 行）：

- **20 个平台适配器**
- 统一会话路由
- 用户授权（白名单 + DM 配对）
- 斜杠命令分发
- Hook 系统
- Cron 调度
- 后台维护任务

### 会话存储

SQLite + FTS5 全文搜索：

- 会话谱系追踪（压缩前后的父子关系）
- 按平台隔离
- 原子写入 + 竞争处理

## 数据流

### CLI 会话流程

```
用户输入 → HermesCLI.process_input()
  → AIAgent.run_conversation()
    → prompt_builder.build_system_prompt()
    → runtime_provider.resolve_runtime_provider()
    → API 调用（chat_completions / codex_responses / anthropic_messages）
    → 检测到 tool_calls? → model_tools.handle_function_call() → 循环
    → 最终回复 → 显示 → 保存到 SessionDB
```

### 网关消息流程

```
平台事件 → Adapter.on_message() → MessageEvent
  → GatewayRunner._handle_message()
    → 用户授权
    → 解析会话键
    → 创建 AIAgent（带会话历史）
    → AIAgent.run_conversation()
    → 通过适配器返回回复
```

### Cron 任务流程

```
调度器触发 → 从 jobs.json 加载到期任务
  → 创建新的 AIAgent（无历史）
  → 注入关联技能作为上下文
  → 执行任务 Prompt
  → 将结果投递到目标平台
  → 更新任务状态和下次执行时间
```

## 设计原则

| 原则 | 实践 |
|------|------|
| **Prompt 稳定性** | 会话中系统 Prompt 不改变，除非用户显式操作（如 /model） |
| **可观测执行** | 每次工具调用对用户可见，CLI 显示进度，网关发消息 |
| **可中断** | API 调用和工具执行可被用户输入或信号中断 |
| **平台无关核心** | 同一个 AIAgent 服务 CLI、网关、ACP、批处理 |
| **松耦合** | 可选子系统使用注册表模式，非硬依赖 |
| **Profile 隔离** | 每个 Profile 独立配置、记忆、会话、网关，可并发运行 |
